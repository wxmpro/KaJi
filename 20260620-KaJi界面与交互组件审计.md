## 0. 审计结论速览

| 分类 | 数量 | 占比 | 评价 |
|------|------|------|------|
| ✅ 已是 macOS 26 最新（或同等） API | 31 处 | ~ 60% | 良好：@Observable、@Environment(T.self)、@Bindable 子作用域、unifiedCompact(showsTitle:)、onChange 两参签名、scrollContentBackground、sizeThatFits(_:nsView:context:)、ContentUnavailableView、NavigationSplitView(.balanced)、windowToolbarStyle 等都已对齐 |
| ⚠️ 不是最新，可升级 | 11 处 | ~ 21% | 需要评估：TabView 充当 Settings 容器、TagPill 用 .primary 分割线、CardType tint 硬编码、contentMargins、shape style 命名、SF Symbols 渲染、window toolbar workaround 残留等 |
| 🔧 自定义实现（合理 / 保留） | 10 处 | ~ 19% | 有正当理由：`NoScrollTextEditor`（SwiftUI TextEditor 无强制行高 API）、`SemanticColor`（统一设计 token）、`KaJi*` 常量体系（项目级 design system）、`draft: DraftState` 状态机（业务域抽象，非 UI API 范畴）、`addUpdateObserver`（@Observable 没有 Combine publisher 的轻量替代） |

> 一句话：**整体已对齐 macOS 26 范式，零过时 API（无 `onChange(perform:)` 旧签名、无 `@StateObject`/`@EnvironmentObject`），但 macOS 26 标志性的 Liquid Glass 完全未引入，是最大缺口。**

## 2. ⚠️ 不是最新 macOS 26 API，可升级

### 2.1 macOS 26 Liquid Glass 完全没有引入（**最大缺口**）

| 现状 | macOS 26 新 API | 证据链接 |
|------|-----------------|----------|
| `SidebarRowButtonStyle`、`KaJiListRowButtonStyle`、`KaJiHoverBackground` 都用 `.background(RoundedRectangle.fill(自定义 Color))` 实现 hover/selected 视觉 | macOS 26 提供 `.glassEffect()` 修饰符，可一行实现 hover 玻璃背景 | <https://developer.apple.com/documentation/swiftui/applying-liquid-glass-to-custom-views> |
| 浮层（CardTypePickerView）背景用自定义 `.background(RoundedRectangle.fill(cardBackground).overlay(...).shadow(...))` | macOS 26 `Glass.regular` + `.glassEffect(.regular.tint(...).interactive())` 系统玻璃 | <https://developer.apple.com/documentation/swiftui/glass/regular>，<https://developer.apple.com/documentation/swiftui/glass/tint%28_%3A%29> |
| toolbar 背景完全没声明材质 | macOS 26 toolbar 推荐 Liquid Glass 材质 | <https://developer.apple.com/documentation/swiftui/landmarks-refining-the-system-provided-glass-effect-in-toolbars> |
| 整窗背景纯 `.background(Color)` | macOS 26 主窗可加 `.glassEffect()` 配合 `Material` 系统材质 | <https://developer.apple.com/documentation/swiftui/material> |

**风险等级**：🟡 中（视觉风格落后 macOS 26 整体设计语言）

### 2.2 `SettingsView` 用 `TabView` 充当 macOS Settings

```swift
// SettingsView.swift:42-57

    generalTab.tabItem { Label("通用", systemImage: "gearshape") }

    aboutTab.tabItem { Label("关于", systemImage: "info.circle") }
```

| 现状 | macOS 26 推荐做法 | 证据链接 |
|------|-------------------|----------|
| macOS Settings 用 `TabView`（顶部 tab + 下方表单），不是 macOS 原生「系统设置」的三列侧栏布局 | macOS 26 推荐 `NavigationSplitView { List(Settings) } detail: { Form }`，或直接用系统 `SettingsLink`/`SettingsScene` 协议 | <https://developer.apple.com/documentation/swiftui/picking-container-views-for-your-content>，<https://developer.apple.com/documentation/swiftui/pickerstyle/tabs>（`.tabs` pickerStyle 是 macOS 26 的原生替代） |

**升级建议**：把三个 tab 改为 `.pickerStyle(.tabs)` 风格，或用 `NavigationSplitView` 三列。

**风险等级**：🟢 低（功能完好，UX 不够原生）

| 理由 | 苹果官方限制 |
|------|---------------|
| SwiftUI `TextEditor` 在 macOS 上**没有**强制行高的 API；kaJi 要让横线对齐文字基线（ruled paper 笔记视觉） | <https://developer.apple.com/documentation/swiftui/texteditor>（26 文档中仅支持 `.font` / `.lineSpacing` / `.scrollContentBackground`，没有行高参数） |

**关联设计 token**：`KaJiMetrics.editorLineHeight = 25`（单一权威值）。

### 3.2 `SemanticColor` 浅深双值 struct

```swift
// SemanticColor.swift
struct SemanticColor: Sendable {
    let light: Color
    let dark: Color
    func resolve(for scheme: ColorScheme) -> Color { ... }
}
```

| 理由 | 苹果官方限制 |
|------|---------------|
| SwiftUI 没有 "一个 token 同时含 light + dark 字段" 的内置类型；想避免调用方写 `scheme == .dark ? x : y` | macOS 13+ 改用 Asset Catalog `Color("name")` 可以同样效果，但项目选 token 化是为了集中管理（见 `semantic-color-token-pattern` memory） |

### 3.3 `KaJiColor` / `KaJiAnimation` / `KaJiLayout` / `KaJiMetrics`

| 理由 | 苹果官方限制 |
|------|---------------|
| SwiftUI 没有项目级命名空间常量；想要一处改全局生效 | 苹果文档没提供；但 macOS 26 推荐方式仍是 Asset Catalog + `Color("name")` + duration 常量集中。KaJi 选了 struct/enum 命名空间（与 Asset Catalog 可并行不冲突） |

### 3.4 `DraftState` 状态机（enum with cases）

```swift
// DraftState.swift
enum DraftState: Equatable {
    case empty(CardType = .free)
    case editing(Card)
    case trash(Card)
}
```

| 理由 | 苹果官方限制 |
|------|---------------|
| SwiftUI 不提供状态机抽象；用 enum cases 让编译器强制处理每个 case 是 v1.4.0 重构后的核心 | <https://docs.swift.org/swift-book/documentation/the-swift-programming-language/enumerations/> |

### 3.5 `StatsState.addUpdateObserver` / `removeUpdateObserver`

```swift
// StatsState.swift:28-37
func addUpdateObserver(_ observer: @escaping () -> Void) -> UUID { ... }
func removeUpdateObserver(token: UUID) { ... }
```

| 理由 | 苹果官方限制 |
|------|---------------|
| `@Observable` 没有 `objectWillChange` publisher；KaJi 需要通知 `ListState` 触发 `refreshFilteredCards`，自己造一个 UUID-token 观察者协议 | <https://developer.apple.com/documentation/combine/observableobject/publisher>（那是 ObservableObject 的 API） |

### 3.6 `KaJiListRowButtonStyle` / `SidebarRowButtonStyle` 自定义 ButtonStyle

| 理由 | 苹果官方限制 |
|------|---------------|
| SwiftUI 内置 `ButtonStyle.plain / .borderless / .bordered / .borderedProminent` 都不能完全控制 hover/selected 颜色阶；KaJi 需要"深浅模式统一、选中/按下/hover 三态"语义 | <https://developer.apple.com/documentation/swiftui/primitivebuttonstyle/configuration>（已查证） |

### 3.7 `kajiHover(cornerRadius:restingBackground:)` ViewModifier

```swift
// KaJiHoverBackground.swift:33-36
func kajiHover(cornerRadius: CGFloat = 6, restingBackground: Color = .clear) -> some View { ... }
```

| 理由 | 苹果官方限制 |
|------|---------------|
| macOS 26 `.glassEffect(.regular.interactive())` 是更现代的方案，但项目目前没用 Liquid Glass；自定义 hover 是合理过渡（与 2.1 节升级机会对应） | 同 2.1 |

### 3.8 `SettingsView` 三个 tab 内部用 `Form { Section { ... } header: { ... } }`

| 理由 | 苹果官方限制 |
|------|---------------|
| 这是 Apple 官方文档推荐的 macOS Settings 容器写法（macOS 13+） | <https://developer.apple.com/documentation/swiftui/form> |

### 3.9 `CardTypePickerView` 内联浮动层

| 理由 | 苹果官方限制 |
|------|---------------|
| SwiftUI `.menu` / `Picker` 提供原生下拉，但 KaJi 要 12 个类型的「彩色圆点 + 文字」网格，自定义更紧凑；自定义代码 40 行 | <https://developer.apple.com/documentation/swiftui/menu>（已是 macOS 14+ 现代 API，但视觉不符需求） |

### 3.10 `KaJiError` / `AppLifecycleError`

| 项目级错误分类；非 UI API，列出仅为完整 | <https://developer.apple.com/documentation/foundation/localizederror> |

## 4. macOS 26 还没用上的新 API（潜在升级机会表）

> 不是"用了错误的 API"，而是"完全没用上 macOS 26 提供的新能力"。

| 新 API | 现状 | 引入价值 | 文档链接 |
|--------|------|----------|----------|
| `.glassEffect()` / `Glass.regular` / `.glassEffect(.regular.tint(.orange).interactive())` | 完全未用 | 🟡 中 — 视觉对齐 macOS 26 设计语言 | <https://developer.apple.com/documentation/swiftui/applying-liquid-glass-to-custom-views> |
| `ToolbarSpacer(.fixed)` / `.flexible` | 完全未用（toolbar 只有 2 个按钮） | 🟢 低 — 当前不需要 | <https://developer.apple.com/documentation/swiftui/toolbarspacer> |
| `.inspector(isPresented:content:)` 第三个右栏 | 未用（KaJi 是 2 栏 NavigationSplitView） | 🟢 低 — 看产品需求 | <https://developer.apple.com/documentation/swiftui/view/inspector%28ispresented%3Acontent%3A%29> |
| `FormStyle.columns` | 当前用 `.grouped`；`.columns` 更"原生" | 🟢 低 | <https://developer.apple.com/documentation/swiftui/formstyle/columns> |
| `.scrollPosition($position)` | 列表无滚动位置记忆 | 🟢 低 — 非核心 | <https://developer.apple.com/documentation/swiftui/scrollposition> |
| `buttonRepeatBehavior(_:)` | 未用 | 🟢 低 — 非核心 | <https://developer.apple.com/documentation/swiftui/buttonrepeatbehavior/disabled> |
| `symbolColorRenderingMode(_:)` / `symbolVariableValueMode(_:)` | 只用基础 `symbolRenderingMode` | 🟢 低 | <https://developer.apple.com/documentation/swiftui/image/symbolrenderingmode%28_%3A%29> |
| `ContentUnavailableView.search` | `CardListView` 用了 `ContentUnavailableView` 但**手写**搜索空态描述（"在「\(listFilterTitle)」下没有卡片"）；可用 `.search` 系统预置 | 🟢 低 | <https://developer.apple.com/documentation/swiftui/contentunavailableview/search> |
| `SearchFieldPlacement.sidebar` | 当前用 `.toolbar` | 🟢 低 — 当前体验更好 | <https://developer.apple.com/documentation/swiftui/searchfieldplacement/sidebar> |
| `LabeledContent` | SettingsView.aboutTab 完全手写 `VStack`；可用 `LabeledContent("版本", value: appVersion)` | 🟢 低 | <https://developer.apple.com/documentation/swiftui/labeledcontent> |
